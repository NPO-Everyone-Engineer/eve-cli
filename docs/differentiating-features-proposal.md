# EvE CLI 差別化機能提案

## 現状分析

EvE CLI は以下の強みを持つローカルファーストの AI コーディングエージェントです：

- **ローカル完結**: Ollama を使用し、データを外部送信しない
- **日本語ファースト**: 日本語ネイティブ対応
- **ログイン不要**: アカウント登録なしで即利用可能
- **マルチエージェント**: 複数エージェントの並列・協調実行
- **単一ファイル配布**: 依存関係なしで配布・実行可能

## 提案する差別化機能

### 1. ローカル音声入力モード (`/voice`)

**概要**: Ollama 互換の音声認識モデル（Whisper ベース）を利用し、声でコーディング指示を出す機能

**差別化理由**:
- クラウド系ツールは音声データを外部送信するため、機密プロジェクトに使えない
- 日本語音声認識の精度を重視したローカル処理
- ハンズフリー開発（ペアプロ、発表中のライブコーディング等）

**実装イメージ**:
```
$ eve-cli --voice
🎤 聞いています... (Ctrl+C で停止)
> 「ユーザー認証モジュールにバリデーションを追加して」
→ 音声をテキストに変換し、通常の指示として処理
```

**技術要件**: `faster-whisper` または Ollama の音声モデル対応

---

### 2. チームメモリ共有 (`/team-memory`)

**概要**: プロジェクト固有の知識・ルール・過去の決定事項をチームで共有・管理する機能

**差別化理由**:
- 現状のメモリは個人ローカルに限定
- チームでの AI 活用時、同じ文脈を全員が持てる
- オンボーディングの大幅短縮（新メンバーが即座にプロジェクト文脈を取得）

**実装イメージ**:
```
.eve-cli/
  team-memory/
    decisions.md      # アーキテクチャ決定記録
    conventions.md    # コーディング規約
    domain-glossary.md # ドメイン用語集
```

**コマンド例**:
```
/team-memory save "認証にはJWTではなくセッションCookieを使う理由"
/team-memory search "認証"
/team-memory sync  # Git経由でチームと共有
```

### 3. 対話型アーキテクチャ図生成 (`/diagram`)

**概要**: コードベースを解析し、クラス図・シーケンス図・依存関係グラフをターミナルまたは SVG で生成

**差別化理由**:
- ローカル LLM でコードを解析するため、機密コードを外部送信しない
- テキストベースのターミナルツールでありながら視覚的理解をサポート
- 日本語コメント・変数名に対応した図生成

**実装イメージ**:
```
/diagram class src/auth/
/diagram sequence "ログイン処理のフロー"
/diagram deps --format=mermaid
```

出力例（ターミナルの場合）:
```
┌─────────────┐     ┌──────────────┐
│  AuthService │────▶│  UserRepository│
└─────────────┘     └──────────────┘
```

### 4. セキュリティ監査エージェント (`/security-audit`)

**概要**: ローカル LLM を使用してコードのセキュリティ問題を自動検出・修正提案する専門エージェント

**差別化理由**:
- クラウド系セキュリティツールはコードを外部送信（機密コードに使えない）
- OWASP Top 10 に特化したプロンプトエンジニアリング
- 日本語での脆弱性説明と修正ガイド

**実装イメージ**:
```
/security-audit
🔍 セキュリティ監査を開始します...

⚠️  [HIGH] SQLインジェクション検出: src/db/query.py:45
   問題: ユーザー入力が直接クエリに埋め込まれています
   修正: パラメータ化クエリを使用してください
   [修正を適用する? y/n]

⚠️  [MEDIUM] ハードコードされたAPIキー: config/settings.py:12
   ...
```

### 5. ペアプログラミングモード (`/pair`)

**概要**: AI が「助手席のプログラマー」として、ユーザーが書いたコードをリアルタイムで説明・改善提案する教育的モード

**差別化理由**:
- 既存の Learning Mode を発展させた双方向教育体験
- ジュニア開発者・プログラミング初学者向けの強力なオンボーディングツール
- 説明は常に日本語でわかりやすく（技術英語の壁を除去）

**実装イメージ**:
```
/pair --explain-level=beginner

👥 ペアプログラミングモード開始
   あなたがコードを書くと、リアルタイムで説明します

[ユーザーが for ループを書く]
💬 「for i in range(10):」を書きましたね！
   これは 0 から 9 まで繰り返す命令です。
   ヒント: リストの要素を直接使う場合は
   「for item in my_list:」の書き方が Python らしいです。
```

### 6. マルチリポジトリコンテキスト (`/workspace`)

**概要**: 複数のリポジトリを横断して、コンテキストを統合した AI エージェントを実行する機能

**差別化理由**:
- マイクロサービス構成や monorepo での開発を強力にサポート
- API の変更が下流サービスに与える影響を自動検出
- 複数リポジトリにまたがるリファクタリングの自動化

**実装イメージ**:
```yaml
# .eve-workspace.yaml
workspace:
  name: "ECサービス群"
  repos:
    - path: ../api-gateway
      role: "APIゲートウェイ"
    - path: ../user-service
      role: "ユーザー管理サービス"
    - path: ../payment-service
      role: "決済サービス"
```

```
/workspace load .eve-workspace.yaml
> 「user-serviceのUserクラスにphone_numberフィールドを追加して、
   影響するサービスも全部更新して」
→ 全リポジトリを横断して変更・PRを作成
```

### 7. プロジェクト健全性ダッシュボード (`/health`)

**概要**: コードベースの品質・技術的負債・テストカバレッジ・セキュリティ状況を日本語でレポートする機能

**差別化理由**:
- CI/CD との連携でなく、ローカルで即座に確認可能
- 数値だけでなく、日本語での改善優先度と具体的アクションを提示
- 経営層・非技術者向けのサマリーレポート生成機能

**実装イメージ**:
```
/health --report

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

総合スコア: 72/100 (普通)

🔴 要対応:
  - テストカバレッジ: 34% (目標: 80%)
  - セキュリティ: 高リスク脆弱性 2件

🟡 改善推奨:
  - 重複コード: 23箇所
  - 技術的負債: 推定 40時間

🟢 良好:
  - ドキュメント率: 89%
  - 依存関係: 最新

📝 今週対応すべき TOP 3:
  1. [CRITICAL] SQLインジェクション修正 (src/db/query.py)
  2. [HIGH] テスト追加 (src/payment/ - 0%カバレッジ)
```

## まとめ

EvE CLI の最大の強みは **「ローカル完結 × 日本語ネイティブ × 無料OSS」** の三点セットです。この強みを活かすために：

1. **機密プロジェクトでも使える安心感**（セキュリティ監査、音声入力）をさらに強化
2. **チーム開発への対応**（チームメモリ、マルチリポジトリ）で個人ツールからの脱却
3. **教育・オンボーディング用途**（ペアプログラミング、健全性ダッシュボード）の強化

これらの機能を組み合わせることで、「企業の機密プロジェクトでも安心して使えるチーム AI コーディングエージェント」というポジショニングが確立できます。
