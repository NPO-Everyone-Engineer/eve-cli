## 入力方法

| 操作 | 説明 |
|------|------|
| **Enter** | 改行（入力を続ける） |
| **空のEnter** | メッセージ送信 |
| **`"""`** | 明示的なマルチラインモード（`"""`で開始・終了） |
| **Tab** | ファイルパス・コマンドの自動補完 |
| **@ファイル名** | ファイル内容を自動添付（例: `@src/main.py を修正して`） |
| **ESC** | エージェントの実行を中断 |

### 画像を添付する方法

```
> /image                          # クリップボードの画像を添付
> /image ~/Desktop/screenshot.png  # ファイルパスで指定
> (ターミナルに画像をドラッグ&ドロップ)
```

> **注意**: 画像を使うにはビジョン対応モデル（`llava`, `llama3.2-vision`, `gemma3` 等）が必要です。

## ループモード（自動再実行）

`--loop` フラグを使用すると、AI が完了シグナルを出力するまでプロンプトを自動で再実行します。
Claude Code の「/loop」機能と同様に、長時間の自律タスクを実行できます。

```bash
# テストが全部通るまで自動で修正を繰り返す
eve-cli -p "pytest を実行して失敗があれば修正してください。全テスト通過したら ALL_DONE と出力してください" \
  --loop --max-loop-iterations 5 --done-string "ALL_DONE" -y

# 最大 3 日間（72 時間）自律実行させる
eve-cli -p "このプロジェクトに認証機能を追加して。完了したら DONE と出力" \
  --loop --max-loop-hours 72 --done-string "DONE" -y
```

| オプション | 説明 | デフォルト |
|-----------|------|-----------|
| `--loop` | ループモードを有効化（`-p` と併用必須） | — |
| `--max-loop-iterations N` | 最大再実行回数 | `5` |
| `--done-string TEXT` | 完了シグナル文字列（AI の応答に含まれると停止） | `"DONE"` |
| `--max-loop-hours N` | 最大実行時間（時間単位、最大 72 時間） | なし |

**注意点**:
- `--loop` は `-p` オプションと併用した場合のみ有効です
- 各反復で会話履歴は引き継がれませんが、ファイル変更は保持されます
- 完了判定は AI の最終テキスト応答のみを対象とします（ツール出力は対象外）
- `--max-loop-hours` を設定すると、時間制限到達時にもループが終了します

## パーミッション（実行許可）

EvE CLI はファイル書き込みやコマンド実行の前に必ず確認を求めます。これにより、意図しない操作を防ぎます。

### 確認画面の選択肢

| 選択肢 | 意味 | 詳細 |
|--------|------|------|
| `y` | 今回だけ許可 | この1回の操作だけ許可します |
| `a` | このツールをずっと許可 | 同じツールの今後の操作をすべて許可します |
| `n` / Enter | 拒否 | この操作を拒否します（デフォルト） |
| `d` | このツールをずっと拒否 | 同じツールの今後の操作をすべて拒否します |
| `Y` | すべてのツールを自動許可 | `-y` モードと同じ（上級者向け） |

### パーミッションの永続化

`a`（すべて許可）や `d`（すべて拒否）を選択すると、その設定は `~/.config/eve-cli/permissions.json` に自動保存されます。次回起動時も設定が引き継がれます。

> **安全のため**: `Bash`、`Write`、`Edit`、`NotebookEdit` の4つは永続的な自動許可ができません（毎回確認されます）。

### Bash コマンドのハイライト表示

確認画面では、危険なコマンド（`sudo`, `rm` 等）が赤色でハイライト表示されるので、実行前に内容を確認しやすくなっています。

## 安全性・復旧機能

EvE CLI は安全性を最優先に設計されています。

### 自動チェックポイント

- `Write` / `Edit` / `MultiEdit` / `NotebookEdit` でファイルを変更する前に、作業状態を自動バックアップ
- バックアップは Git の stash を使って非破壊的に保存されます

### 復旧方法

```
> /rollback    # 最新のチェックポイントに戻す
> /undo        # 直前のファイル変更だけ元に戻す
```

### 状態確認

```
> /status
```

以下の情報が表示されます:
- セッション情報（モデル、トークン使用量）
- Git の状態（変更ファイル数、最近の変更）
- チェックポイント数
- 自動テストの設定状態

### リッチdiffプレビュー

ファイルを編集する際、変更前後の差分が詳細に表示されます:
- **赤色（`-`）**: 削除される行
- **緑色（`+`）**: 追加される行
- 最大6行まで表示、それ以上は行数サマリーを表示

## 自動コンパクション（会話の自動圧縮）

長い会話を続けていると、AIが覚えられるコンテキストの上限に近づきます。EvE CLI はこれを自動で管理します。

### 仕組み

1. トークン使用量が **コンテキストウィンドウの80%** を超えると自動発動
2. 古い会話をAIが要約し、コンパクトに圧縮
3. 直近の4メッセージは完全に保持
4. 5イテレーションごとにチェック

### 手動コンパクション

```
> /compact    # 今すぐ会話を圧縮
```

> **ヒント**: 長時間のコーディングセッションでも、自動コンパクションのおかげで途中でリセットする必要がありません。

## エラー自動修復

コマンドやテストが失敗した場合、EvE CLI は自動でエラーを分析し、修正を試みます。

### 仕組み

1. `Bash`、`Edit`、`Write` ツールがエラーを返した場合に発動
2. エラー内容をAIに自動フィードバック
3. AIが原因を分析し、次のステップで修正を実行
4. **安全制限**: 同じエラーに対する自動修復は最大2回まで（無限ループ防止）

### 例

```
> Pythonスクリプトを作って実行して

  ✘ Bash → python3 app.py
  (SyntaxError が発生)

  [AIが自動でエラーを分析し、ファイルを修正して再実行]

  ✔ Bash → python3 app.py
  (正常に実行完了)
```

## テスト生成

AIが指定したファイルのユニットテストを自動生成します。

### 使い方

```
> /gentest src/calculator.py
```

以下の処理が自動で行われます:

1. ソースファイルの読み込み
2. 既存テストファイルがあればスタイルを参照
3. AIがユニットテストを生成
4. `tests/test_calculator.py` として保存
5. 構文チェックを自動実行

### 特徴

- 既存テストのスタイル（命名規則、アサーション形式等）を自動的に踏襲
- エッジケースやエラーケースも含む包括的なテスト
- `tests/` ディレクトリと `__init__.py` を自動作成
- 既存ファイルがある場合は上書き確認

## コミットメッセージ自動生成

`/commit` コマンドで、AIが差分を分析してコミットメッセージを自動提案します。

### 使い方

```
> /commit
```

### 仕組み

1. `git diff --cached` で差分を取得
2. `git log --oneline -5` で直近のコミットスタイルを参照
3. AIが [Conventional Commits](https://www.conventionalcommits.org/) 形式でメッセージを生成
4. プロジェクトの言語（日本語/英語）に自動対応
5. 確認画面でメッセージを編集・承認

### 例

```
> /commit

  Generated commit message:
  feat(auth): ログイン画面にパスワードリセット機能を追加

  [y] Commit  [e] Edit  [n] Cancel
```

> **フォールバック**: AIが応答できない場合は、変更ファイル名から自動的にメッセージを生成します。

## GitHub連携

[GitHub CLI (`gh`)](https://cli.github.com/) がインストールされていれば、EvE CLI から直接プルリクエストの管理ができます。

### プルリクエスト操作

```
> /pr                  # PR一覧を表示
> /pr create           # 新しいPRを作成（AIがタイトル・説明を提案）
> /pr 42               # PR #42 の詳細を表示
> /pr checks 42        # CIチェック状況を確認
> /pr diff 42          # 差分を表示
> /pr merge 42         # PRをマージ
```

### GitHub CLIコマンド

```
> /gh issue list            # Issue一覧
> /gh release list          # リリース一覧
> /gh repo view             # リポジトリ情報
```

> **前提条件**: `gh` コマンドがインストール済みで `gh auth login` 完了済みであること。
