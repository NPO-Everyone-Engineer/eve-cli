## 基本操作

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプを表示 |
| `/clear` | 会話をクリアして最初から始める |
| `/compact` | 会話を手動で要約・圧縮する |
| `/config` | 現在の設定を表示 |
| `/model` | 使用モデルを切り替え（対話式） |
| `/model <名前>` | 指定したモデルに直接切り替え |
| `/tokens` | トークン使用量を表示 |
| `/exit` | EvE CLI を終了（Ctrl+D でも可） |

## Plan/Act モード

「まず計画を立て、承認してから実行」する安全な開発フローです。
Plan モード中は通常ファイル編集や write-capable sub-agent がブロックされ、書き込みは `.eve-cli/plans/` のみ許可されます。

| コマンド | 説明 |
|---------|------|
| `/plan` | Plan モードに切り替え（読み取り専用で計画を立てる。書き込みは `.eve-cli/plans/` のみ） |
| `/approve` | 計画を承認して Act モードで実行 |

**使い方の例：**

```
> /plan
> ログイン機能を追加したい
（AI が計画を提示）
> /approve
（AI が計画に沿って実装を開始）
```

## コードインテリジェンス

プロジェクト内の関数・クラスを高速検索する機能です。

| コマンド | 説明 |
|---------|------|
| `/index build` | シンボルインデックスを構築（初回のみ） |
| `/index search <クエリ>` | 関数名・クラス名を検索 |
| `/index file <パス>` | ファイル内のシンボル一覧を表示 |
| `/index status` | インデックスの状態を確認 |

対応言語: Python, JavaScript/TypeScript, Go, Rust, Ruby, Java

## GitHub 連携

[GitHub CLI (`gh`)](https://cli.github.com/) がインストール済みの場合に使えます。

| コマンド | 説明 |
|---------|------|
| `/pr` | プルリクエスト一覧を表示 |
| `/pr create` | 新しい PR を作成（AI がタイトル・説明を提案） |
| `/pr <番号>` | 指定 PR の詳細を表示 |
| `/pr checks <番号>` | CI チェック状況を確認 |
| `/pr diff <番号>` | 差分を表示 |
| `/pr merge <番号>` | PR をマージ |
| `/gh <コマンド>` | GitHub CLI コマンドを直接実行 |

## マルチエージェント

複数の AI エージェントに大きなタスクを分担させます。

| コマンド | 説明 |
|---------|------|
| `/team <ゴール>` | Agent Team を起動 |
| `/team -n <名前> -w <人数> <ゴール>` | チーム名とワーカー数を指定 |

**例：**

```
> /team このプロジェクトにログイン機能を追加して
> /team -n "auth-team" -w 3 認証の設計・実装・テストを行って
```

## Channels

Discord・Slack・Webhook から実行中のセッションにメッセージを送受信する機能です。
詳細なセットアップ手順は [高度な機能 → Channels](advanced.md#channels外部チャンネル連携) を参照してください。

### 共通

| コマンド | 説明 |
|---------|------|
| `/channels list` | 接続中のチャンネルと状態を表示 |
| `/channels stop` | 全チャンネルを停止 |

### Webhook

| コマンド | 説明 |
|---------|------|
| `/webhook:configure [APIキー]` | API キーを設定して認証を有効化 |
| `/webhook:allow <送信者ID>` | 指定した送信者を許可 |
| `/webhook:allowlist` | 許可済み送信者の一覧を表示 |
| `/webhook:open` | 全員を許可（開発・テスト用のみ） |

### Discord

| コマンド | 説明 |
|---------|------|
| `/discord:configure <トークン> [チャンネルID...]` | ボットトークンとチャンネル ID を設定 |
| `/discord:pair <コード>` | ペアリングコードで送信者を認証・許可 |
| `/discord:access policy allowlist` | 許可済みユーザーのみ（デフォルト） |
| `/discord:access policy open` | 全員を許可（開発・テスト用のみ） |

### Slack

| コマンド | 説明 |
|---------|------|
| `/slack:configure <トークン> <署名シークレット>` | ボットトークンと署名シークレットを設定 |
| `/slack:allow <ユーザーID>` | 指定した Slack ユーザーを許可 |
| `/slack:access policy allowlist` | 許可済みユーザーのみ（デフォルト） |
| `/slack:access policy open` | 全員を許可（開発・テスト用のみ） |

## AI ツール一覧

AI が自動的に選んで使うツールです。ユーザーが直接呼び出す必要はありません。

### 読み取り系（自動許可）

| ツール | 説明 |
|-------|------|
| **Read** | ファイルを読む（テキスト、画像、PDF、Jupyter 対応） |
| **Glob** | ファイル名のパターン検索 |
| **Grep** | ファイル内容のテキスト検索 |
| **SubAgent** | サブエージェントを起動して調査・分析 |
| **ParallelAgents** | 複数のサブエージェントを並列実行 |
| **TaskCreate / TaskList / TaskGet / TaskUpdate** | タスク管理 |
| **AskUserQuestion** | ユーザーに質問する |

### 書き込み・実行系（確認が必要）

| ツール | 説明 |
|-------|------|
| **Bash** | シェルコマンドを実行 |
| **Write** | ファイルを新規作成 |
| **Edit** | ファイルの一部を編集 |
| **MultiEdit** | 複数ファイルを一括編集（最大 20 ファイル） |
| **WebFetch** | URL から Web ページを取得 |
| **WebSearch** | Web 検索 |
| **NotebookEdit** | Jupyter Notebook のセル編集 |
| **MCP** | MCP サーバー経由の外部ツール |

> 「確認が必要」なツールは、実行前に必ず許可を求めます。`-y` オプションで起動すると確認をスキップできます（上級者向け）。
