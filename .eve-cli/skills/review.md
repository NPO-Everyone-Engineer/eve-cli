# コードレビュー (Code Review)

## 概要
コードを包括的にレビューし、改善点を提案します。

## 使用方法
```
/custom review [ファイル名またはディレクトリ]
```

## レビュー項目

### 1. セキュリティ問題 🔒
- ハードコードされたシークレット（API キー、パスワード等）
- SQL インジェクションのリスク
- パストラバーサルの脆弱性
- 入力値の検証不足
- 不適切なアクセス制御

### 2. パフォーマンス問題 ⚡
- 非効率的なループ処理
- 不要なメモリアロケーション
- データベースクエリの非最適化
- 重複計算
- 不適切なデータ構造の使用

### 3. コードスタイル 📝
- プロジェクトのコーディング規約への準拠
- 命名規則の一貫性
- コードの可読性
- コメントの適切さ
- 関数/メソッドの長さ

### 4. エラーハンドリング ⚠️
- 例外処理の適切さ
- エラーメッセージの明確さ
- エッジケースの考慮
- リソースの適切な解放
- ログ出力の適切さ

## 出力形式

- 具体的な行番号を記載
- 問題の深刻度（高/中/低）
- 改善提案とコード例
- 参考資料（該当する場合）

## 使用例

```bash
# 現在のディレクトリをレビュー
/custom review

# 特定のファイルをレビュー
/custom review src/auth.py

# 特定のディレクトリをレビュー
/custom review src/components/
```

## レビュー対象ファイル

- 最近変更されたファイル（git diff）
- 重要なコアファイル
- 指定されたファイル/ディレクトリ

## ヒント

- 特定の焦点（セキュリティのみ等）を指定可能
- 複数回のレビューで異なる観点から確認可能
- PR 作成前の最終確認に推奨
