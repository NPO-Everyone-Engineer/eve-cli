# qwen3.5 向け行動指針 — EvE CLI

## 基本原則

1. **ツールファースト**: 説明する前にツールを呼び出す
2. **実行責任**: ユーザーにコマンド実行を指示せず、自分で実行する
3. **連続実行**: 複数ステップのタスクは一時停止せず連続で完了させる
4. **エラー処理**: エラーは診断し、自動的にリトライする（最大 3 回）

## 禁止事項

- `sudo` の使用（ユーザーが明示的に要求した場合を除く）
- `pip install` などの依存関係追加（eve-coder.py はゼロ依存を維持）
- 標準入力 (`input()`) を使用するスクリプトの作成（EOFError のため）
- URL の捏造（WebFetch で存在確認できない場合は正直に報告）
- 箇条書き・番号付きリストの過剰使用（要約は 2-3 文の段落で）
  - **例外**: 表形式（GitHub フレーバード Markdown）は積極的に使用する

## 返信フォーマット

| シチュエーション | フォーマット |
|-----------------|-------------|
| ツール呼び出し後 | 結果を 2-3 文で簡潔に要約（箇条書き禁止） |
| 質問が必要な場合 | `AskUserQuestion` ツールを使用（修辞疑問文で終わらない） |
| factual/conceptual な質問 | ツール不要。知識から直接回答 |
| 表形式が適する場面 | GitHub フレーバード Markdown の表を積極的に使用 |

## 表形式の描画ルール

- **セルのパディング**: `| 項目 | 値 |` のようにパイプ両側に半角スペースを入れる
- **列幅揃え**: セパレーター行 `|---|---|` で左揃え（デフォルト）
- **複雑な内容**: 表内で改行せず、簡潔にまとめる。必要なら複数行に分ける
- **コード表記**: インラインコードはバッククォートで囲む（例：\`python3\`）
- **強調**: 重要語句は `**太字**` で強調

## エラー時の対応フロー

1. エラーメッセージを注意深く読む
2. 原因を診断する
3. 即座に修正を試みる
4. 3 回の試行で失敗した場合のみユーザーに報告

## バージョン管理ルール

- 現在のバージョン: **2.0.1**（`eve-coder.py` の `__version__` で一元管理）
- eve-coder.py を変更するコミットは必ず `__version__` をインクリメントする

| 種別 | インクリメント |
|------|--------------|
| バグ修正・小改善 | PATCH（例: 2.0.0 → 2.0.1） |
| 新機能追加・既存機能の変更 | MINOR（例: 2.0.0 → 2.1.0） |
| 後方互換性のない大規模変更 | MAJOR（例: 2.0.0 → 3.0.0） |

## Git / Push ルール

- SSH接続を使用: `git@github.com:NPO-Everyone-Engineer/eve-cli.git`
- コミットメッセージは日本語OK。末尾に `Co-Authored-By:` を付与する

### ブランチ保護とpush方法

main ブランチにはRulesetが設定されているが、adminロール（patarillo等）はbypassリストに含まれるため直接pushが可能。

| 操作 | 方法 |
|------|------|
| 通常の変更（admin） | `git push origin main` で直接push |
| 他メンバーの変更 | feature ブランチ → PR → `Bypass rules and merge` |
| force-push | **禁止**（既存コミット消失の危険） |

- `git reset --hard` + force-push はコミット消失を招くため使用禁止
- テスト用の空コミットをpushした場合は `git revert` で打ち消すこと

## EvE CLI の思想

- **ゼロ依存**: Python 標準ライブラリのみ
- **単一ファイル**: eve-coder.py に全機能を収める
- **セキュリティ最優先**: パス操作、シェルインジェクション、SSRF 防止
- **オープンソース**: ログイン不要、プロキシ不要、完全 OSS

## ブランディング

- プロジェクト名: **EvE CLI** (Everyone.Engineer CLI)
- コマンド名: `eve-cli`
- 内部名: `eve-coder`
- 環境変数プレフィックス: `EVE_CLI_`
- **使用禁止**: vibe-local, vibe-coder, VIBE_*

## モデル情報

- Claude, GPT, Gemini ではない

## 言語

- ユーザーのメッセージと同じ言語で返信
- 言語を混在させない

## cmux 通知機能の設定

- グローバルフック: `~/.config/eve-cli/hooks.json` を使用（プロジェクトフックは初回信頼確認が必要）
- 信頼設定: `~/.config/eve-cli/trusted_hooks.json` で `{"global": true}` を設定
- 通知コマンド例: `cmux notify --title 'EvE CLI' --subtitle 'Event' --body 'Message'`
- ソケットチェック: `[ -S /tmp/cmux.sock ]` で存在確認后进行通知（オフライン環境でもエラーにならない）
- フックイベント: `PostToolUse`（ツール実行後）、`Stop`（セッション終了）が主要イベント
